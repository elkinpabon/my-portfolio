Sobre el Proyecto

Plataforma SaaS para la gestión de publicaciones académicas y libros, orientada a librerías y bibliotecas. Usa una arquitectura de microservicios para garantizar escalabilidad y flexibilidad en su implementación, con un frontend en React.

Stack de desarrollo: El backend fue desarrollado con Spring Boot utilizando PostgreSQL como base de datos principal, garantizando robustez y consistencia en las transacciones. El frontend está construido con React para proporcionar una experiencia de usuario moderna y reactiva.

Tecnologías de arquitectura: RabbitMQ fue implementado para mensajería asíncrona entre microservicios. Eureka proporciona service discovery automático, mientras que Kubernetes se utiliza para el despliegue y orquestación de contenedores en producción.

Características Implementadas

  • ✅ Gestión completa de libros digitales
  • ✅ Sistema de suscripciones empresariales
  • ✅ Arquitectura de microservicios escalable
  • ✅ Despliegue automatizado con Kubernetes
  • ✅ Comunicación asíncrona entre servicios

Equipo de desarrollo: Proyecto desarrollado por 3 personas especializadas en backend, frontend y DevOps.

Tiempo de desarrollo: El proyecto fue completado en un período de 2 meses, incluyendo la implementación completa de la arquitectura de microservicios y despliegue en Kubernetes.

🚀 Stack Tecnológico Avanzado

Frontend: React 18 con TypeScript, Material-UI para componentes, Redux Toolkit para gestión de estado, React Query para cache y sincronización de datos, y React Router para navegación SPA.

Backend: Node.js con Express, arquitectura de microservicios, GraphQL con Apollo Server, JWT para autenticación, Socket.io para notificaciones en tiempo real, y Redis para cache distribuido.

Base de datos: MongoDB con Mongoose ODM, diseño optimizado para multitenancy, indexación avanzada para búsquedas rápidas, y backup automático diario.

Infraestructura: AWS EC2 con Load Balancer, S3 para almacenamiento de archivos, CloudFront CDN para distribución global, Stripe Connect para pagos multivendor, Docker para containerización, y CI/CD con GitHub Actions.
